# OpenQP/ddX PCM energy and Fock convention

Status: HF/DFT stabilization note for `feat/solvent-backend-spike`.

## Scope

This note documents the current ddX-backed equilibrium ddPCM convention used by
the OpenQP single-point SCF path:

```
pcm_enabled -> add_pcm_reaction_field -> e_pcm -> ddX C adapter
```

It applies to the current HF/DFT ground-state smoke tests only. It does not claim
validated gradients, TDDFT, MRSF, nonequilibrium solvation, or state-specific
excited-state PCM.

## Source convention

The QM PCM source is the atom-centered real-solid-harmonic multipole source
through `l=2` constructed from the current SCF density:

- net atomic charge from nuclear charge minus Mulliken population,
- Mulliken-partitioned dipole moments,
- Mulliken-partitioned quadrupole moments.

The same multipole source is passed to both ddX calls:

- `ddx_multipole_electrostatics` to form the multipole-derived cavity potential
  diagnostic `phi_source`, and
- `ddx_multipole_psi` to build the ddX right-hand side.

The Tier-1 point-charge smoke test is the only path that intentionally retains a
nuclear/point monopole source.

## Fock sign and scale

A fixed-density diagnostic perturbs the cavity potential `phi_cav` while keeping
the same multipole `psi` source fixed, recomputes the ddX energy, and estimates
`dE_ddX/d(phi_cav_i)` for high-|q_cav| cavity points.

For ddX v0.8.0 with the current OpenQP adapter, the observed relation is

```
dE_ddX / d(phi_cav_i) = -0.5 * q_cav_i
```

Therefore the AO reaction-field operator inserted into the SCF Fock build is

```
V_PCM = -0.5 * external_charge_potential(q_cav)
```

The test suite asserts the runtime diagnostics:

- `fock_q_scale = -0.5`,
- `fd_fock_scale_mean ~= -0.5`,
- `fd_fock_scale_rms ~= 0.5`,
- `fd_fock_scale_maxerr < 1e-5`.

## Reported energy convention

`PCM solvent energy` and diagnostic `e_pcm` are the scalar energy returned by the
ddX solve for the current density/source. This is the value added to the printed
OpenQP total energy in the current spike.

`half_tr_dv` is a diagnostic computed from the AO reaction-field potential after
applying the Fock scale above. It is not currently used as the authoritative
reported solvation energy. It is kept in the log to expose any future convention
regression or refactor.

Do not infer correctness from `e_pcm` alone. The required internal checks are:

1. `q_cav_sum` is not the nuclear charge and `q_cav_absnorm` is nonzero.
2. ddX multipole-derived `phi_source` agrees with exact OpenQP `phi_cav` at the
   cavity points within the regression thresholds.
3. The fixed-density finite-difference derivative recovers `-0.5*q_cav`.
4. PCM-off vacuum energies remain unchanged.

## Current diagnostic snapshot

The current identical-protocol OpenQP/ddX diagnostic snapshot is generated by:

```
python scripts/pcm_trusted_reference_diagnostics.py \
  --json-out tests/data/pcm_trusted_reference_diagnostics.json
```

The snapshot records HF and DFT smoke rows but is not a literature benchmark and
must not be promoted to `reference_value` until an independent same-protocol
reference exists.

Current benchmark coverage:

- 10 neutral closed-shell molecules: H2O, NH3, HF, CH4, CO, CO2, HCN,
  H2CO, CH3OH, and CH3CN.
- 3 methods for each geometry: RHF/HF, RHF/BHHLYP, and RHF/PBE with
  `6-31g*` and ddPCM water.

The benchmark table now stores OpenQP/ddX closed-shell trusted-regression
reference values for all 30 rows. These references are hard pass/fail regression
targets for the stabilized diagnostic convention; they are still not literature
benchmarks or independent physical references.

The an independent ddPCM reference vacuum validation snapshot is stored in
`tests/data/pcm_vacuum_pyscf_validation.json`. HF rows are the strict cross-code
gate. DFT rows are protocol-near because OpenQP's LibXC+SG1/MHL grid path is not
bit-identical to the an independent ddPCM reference grid used by the validation script.
